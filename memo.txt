Node.js
- 브라우저 밖에서도 자바스크립트를 실행할 수 있도록 만든 서버 측 자바스크립트 실행 환경
- 구글의 v8 엔진을 기반으로 동작
- 웹 서버, API 서버, 실시간 채팅, 스트리밍 서비스 등 활용
- npm(Node Package Manager)을 통해 수많은 오픈소스 패키지를 쉽게 사용할 수 있어 빠른 개발이 가능
- 프론트엔드와 동일한 언어(JavaScript)를 사용한다는 점에서 풀스택 개방에 매우 유리한 환경

프론트엔드, 백엔드 역할 차이
- 프로트엔드는 사용자가 직접 보고 상호작용하는 화면(UI)을 구현하는 영역. HTML/CSS/JavaScript를 사용해 버튼, 입력창, 화면 레이아웃, 애니메이션 등 사용자 경험을 담당
- 백엔드는 사용자의 요청을 받아 데이터를 처리하고, 데이터베이스와 연결하여 저장/조회/수정/삭제 등의 로직을 수행하는 서버 측 영역

패키지 매니저
- Npde.js 환경에서 필요한 라이브러리(패키비)를 설치, 관리, 업데이트할 수 있게 도와주는 도구
- npm(Node Package Manage): node.js 설치 시 자동으로 같이 설치됨

package.json
- Node.js 프로젝트의 정보를 담고 있는 핵심 파일
- 패키지에 대한 설명서(프로젝트 이름, 버전, 필요한 패키지 목록, 실행 스크립트 등 다양한 정보를 담고 있음)
- 

{
  "name": "07_nodejs",      //  프로젝트 이름(소문자로 작성, 공백 X)
  "version": "1.0.0",       // 프로젝트 버전
  "description": "",        // 긴딘힌 설명
  "main": "index.js",       // 사용 라이선스
  "scripts": {              // 만ㄷ느 사람 이름
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "repository": {
    "type": "git",          // 프로젝트 방식  (강사는 "commonjs")
    "url": "git+https://github.com/kjalh/07_nodejs.git"
  },
  "author": "",
  "license": "ISC",
  "type": "commonjs",
  "bugs": {
    "url": "https://github.com/kjalh/07_nodejs/issues"
  },
  "homepage": "https://github.com/kjalh/07_nodejs#readme"
}


웹서버
- 사용자의 브라우저 요청을 받아 웹페이지, 이미지,  css, JavaScript 같은 파일을 전달해주는 서버 프로그램 또는 시스템
- 사용자가 주소(URL)를 입력하면 웹서버는 해당 요청을 처리하고 필요한 파일이나 데이터를 응답으로 보내며, 이 과정을 통해 브라우저 웹사이트 화면이 표시됨


모듈
- 모듈은 프로그램을 기능별로 나누어 독립적인 파일이나 단위로 구성한 것으로, 코드의 재사용성과 유지보수를 쉽게 해주는 구조
- 특정 기능(계산, 인증, 데이터 처리 등)을 하나의 파일로 분리해두면, 다른 파일에서 필요할 때 불러와 사용할 수 있음
- require() 또는 import를 통해 모듈을 가져올 수 있음


CommonJA vs ES6 모듈 차이
commonjs 
    -node.js기본 방식
    - 모둘 가져오기: require('파일')
    - 내보내기:module.exports 또는 exports.이름
    - 확장자: .js

ES6   지원하는 브라우저 또는 node.js 환경
    모듈 가져오기: imprt{이름}from '파일'
    - 모듈 내보내기 export, export default
    - 확장자: .mjs


MongoDB
- NoSQL 데이터베이스 시스템 중 하나로, 문서 지향 데이터베이스 (진짜 문서가 아님, JS스타일)
- 관계형 데이터베이스와 다르게 데이터를 테이블이 아니라 JSON 스타일의 BSON(Binary JSON) 형식의 문서로 저장
- 개발자가 유연하게 데이터를 저장하고 쿼리할 수 있도록 목적으로 만들어짐

문서 지향 데이터베이스
데이터를 BSON 형식의 문서로 저장. 이 문서는 키-값 쌍으로 이루어져 있으며, 여러 종류의 데이터 유형을 포함할 수 있음

특징
- 스키마 없음
  관계형 데이터베이스와 달리 MongoDB는 데이터베이스의 스키마를 명시적으로 정의하지 않음
  동적인 스키마르 사용하여 데이터 모델을 유연하게 변경할 수 있도록 함

  {"no":1, "userid":"apple", "userpw":"1111"}
  {"no":1, "userid":"banana", "ueserpw":"2222", "email":"banana@banana.com"}
  {"name":오렌지, "hp":"010-1111-1111"}
